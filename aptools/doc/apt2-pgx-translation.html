Introduction

apt2-pgx-translation engine converts genotype calls for markers of pharmacogenetic interest into Ref/Var call nomenclature, reports haplotype pair names for some genes, and translates the haplotype pairs to predicted enzyme phenotypes. This engine replaces apt-dmet-translation.

Quick Start

The command below is formatted for Linux environment and shows typical usage for a PharmacoScan_96F array with the r9 library package. BATCHDIR is the path to the batch folder created by Axiom Analysis Suite using either the Genotyping or Best Practices workflows. The batch folder can also be created by apt-package-util. AXIOM_LIB_PATH is the path to the PharmacoScan_96F.r9 library folder. 

BASENAME=myresults
apt2-pgx-translation \
--batch-folder $BATCHDIR \
--cn-region-calls-file $BATCHDIR/CNData/AxiomCNVMix.cnregioncalls.txt \
--marker-list-file $BATCHDIR/SNPolisher/Recommended.ps \
--analysis-files-path $AXIOM_LIB_PATH \
--translate-file PharmacoScan_96F.r9.na36.translation \
--metabolizer-file PharmacoScan_96F.r9.metabolizer \
--annotation-file PharmacoScan_96F.r9.dc_annot.csv \
--use-first-dup-allele-def true \
--out-dir $BATCHDIR/Translation \
--base-name-prefix $BASENAME \
--log-file $BATCHDIR/Translation/$BASENAME.log

To perform allele translation on a subset of samples in the batch, supply the optional --sample-filter-file argument, pointing to the file containing a list of cel filenames (no path). The first row must be ‘cel_files’. An example file is shown below:

cel_files
Control1_G12.CEL
Control2_H12.CEL
