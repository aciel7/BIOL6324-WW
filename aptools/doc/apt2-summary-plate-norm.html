Introduction

apt2-summary-plate-norm program minimizes between-plate effects by adjusting the signals for some probesets for entire plates. Multiplate normalization performs a linear regression on the summary intensities over the different plates and then shifts each plateâ€™s signals toward a baseline plate to minimize the plate effect. The baseline plate is selected from the plates with the most samples.

Multiplate normalization is used to improve genotyping performance of probesets that originally failed SNP QC in multiple plate analysis. A required input is the Ps.performance.txt file created by ps-classification program on the same analysis used to create the input summary file. Using this input, biallelic probesets are selected for signal adjustment if they belong to a ConversionType of CallRateBelowThreshold, Other, OTV, or UnexpectedGenotypeFreq.

The program will perform multiplate normalization if the input data includes at least two plates or plate scans of at least min-plate-samples each, and at least one probeset is selected.

The output summary file is of the same type (.txt or .a5) as the input summary file, and includes only the multiplate normalized probesets. Other outputs include a list of renormalized probesets, and a platemap report identifying to which plate group each sample was assigned and whether renormalization was applied. The output summary file can be genotyped using apt-summary-genotype-axiom program.

Quick Start

A basic usage example is shown below.

apt2-summary-plate-norm \
  --input-summary-file AxiomGT1.summary.txt \
  --plate-map-file AxiomGT1.report.txt \
  --ps-performance-file Ps.performance.txt \
  --out-dir ./PlateNorm

If the original genotyping run created a rare het report, it should also be supplied to exclude adjusted probesets from renormalization. Otherwise, regenotyping of the output summary file may negate the benefits of the rare het adjustment algorithm. The next example also uses the summary.a5 input format, and allows the program to consider renormalization of biallelic subset probesets of multiallelic probeset designs.

apt2-summary-plate-norm \
  --input-summary-file AxiomGT1.summary.a5 \
  --plate-map-file AxiomGT1.report.txt \
  --rare-het-report-file AxiomGT1.rare_het.report.txt \
  --ps-performance-file Ps.performance.txt \
  --out-dir ./PlateNorm \
  --exclude-multisnp-biallele false
