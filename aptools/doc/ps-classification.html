Introduction

ps-classification reads in the SNP QC metrics table generated by ps-metrics and performs SNP classification based on customizable criteria, including call rate, FLD, HetSO, and HomRO. It classifies SNPs/probesets into seven major categories: "PolyHighResolution","NoMinorHom", "MonoHighResolution", "OffTargetVariant", "CallRateBelowThreshold", "Other", and "UnexpectedGenoFreq". The category "Hemizygous" appears only when hemizygous SNPs are in the metrics file but the special SNPs data is not present in the metrics file. `ps-classification` also identifies the best probeset per SNP.

Quick Start

This command runs ps-classification on a metrics file named metrics.txt in the directory ./input for a human dataset and writes the classification output to the directory ./classification: 

ps-classification --species-type human --metrics-file ./input/metrics.txt --output-dir ./classification

Details

The input files and options for ps-classification are:

  • metrics-file - The name of the file containing output generated by ps-metrics.
  • multi-metrics-file - The name of the multiallelic metrics file generated by ps-metrics.
  • ps2snp-file(Optional) - The name of a file containing the mapping of probeset IDs to SNP IDs. The column names should be "probeset_id" and "snpid". The user must supply this file when classifying de novo SNPs or any SNPs that have more than one probeset.
  • output-dir - Output directory for results files: defaults to current directory.
  • pid-file- The list of probesets that `ps-classification` should run on.

The classification thresholds accepted by ps-classification are:

  • species-type - The type of organism genotyped. The current options are "human", non-human "diploid", and "polyploid".
  • output-recommended - Logical value indicating whether to output a list of recommended SNPs to recommended.ps.
  • recommended - List of categories whose SNPs will be output as recommended. If output.recommended is TRUE, the default value for Human is PolyHighResolution, MonoHighResolution, NoMinorHom (legacy data includes Hemizygous); for Diploid the default is PolyHighResolution, MonoHighResolution, NoMinorHom; and for Polyploid the default is PolyHighResolution.

If a biallelic metrics file is provided, the following classification thresholds are used:

  • cr-cutoff - Threshold for call rate
  • fld-cutoff - Threshold for FLD
  • het-so-cutoff - Threshold for HetSO
  • het-so-otv-cutoff - Threshold for OTV detection
  • hom-ro-1-cutoff - Threshold for HomRO when a SNP has one genotypes
  • hom-ro-2-cutoff - Threshold for HomRO when a SNP has two genotypes
  • hom-ro-3-cutoff - Threshold for HomRO when a SNP has three genotypes
  • hom-ro - Flag indicating whether the metric HomRO is used in classification
  • hom-het - Flag indicating whether the metric HomHet is used in classification. The HomHet metric identifies two-cluster SNPs/probesets with one homozygote cluster and one heterozygote cluster. This checks if the minor homozygote cluster is missing, which is unreasonable for highly inbred species (e.g. wheat). This metric should be turned on when classifying SNPs/probesets in highly inbred species.
  • num-minor-allele-cutoff - Threshold for the number of minor alleles.
  • priority-order - When performing probeset selection, the best probeset is selected according to the priority order of probeset conversion types. Valid values are sequences of the strings "PolyHighResolution", "NoMinorHom", "OTV", "MonoHighResolution","CallRateBelowThreshold", and "UnexpectedGenoFreq", separated by commas, with each string occuring exactly once: default `'PolyHighResolution,NoMinorHom,OTV,MonoHighResolution,CallRateBelowThreshold'`
  • genotype-p-value-cutoff - Minimum p-value for a probeset to be recommended instead of UnexpectedGenotypeFrequency

The flag hom-ro should be turned off for organisms where the homozygote clusters may center at zero in the contrast space, particularly polyploid species.

If the biallelic metrics file contains copy-number aware metrics, the following classification thresholds are used with SNPs with copy-number aware calls:

  • hom-ro-hap-1-cutoff - Threshold for HomRO when a SNP has one haploid genotype
  • hom-ro-hap-2-cutoff - Threshold for HomRO when a SNP has one haploid genotype
  • hom-hap-X-cutoff - Maximum distance diploid clusters can be outside of haploid clusters in the X dimension
  • hom-hap-Y-lower-cutoff - Maximum distance diploid clusters can be below haploid clusters in the Y dimension
  • hom-hap-Y-upper-cutoff - Maximum distance haploid clusters can be above diploid clusters in the Y dimension
  • CN0-hap-X-cutoff - Maximum distance CN0 clusters can be outside of haploid clusters in the X dimension
  • CN0-hap-Y-cutoff - Maximum distance CN0 clusters can be above haploid clusters in the Y dimension
  • CN0-dip-X-cutoff - Maximum distance CN0 clusters can be outside of diploid clusters in the X dimension
  • CN0-dip-Y-cutoff - Maximum distance CN0 clusters can be above diploid clusters in the Y dimension

If the supplemental filter is turned on, the following classification thresholds are used with SNPs with supplemental metrics:

  • use-supplemental - Flag indicating if the supplemental classification should be performed
  • homfld-cut - Minimum HomFLD value for a probeset to be PHR instead of HomHomResolution
  • n-minor-hom-cut - Minimum number of samples in the minor homozygous cluster to be recommended instead of nMinorHom
  • hetv-maf-cut - Cut-off for het rate versus MAF for UnexpectedHeterozygosity
  • variance-class - Category types that the supplemental workflow is run on. Takes a single category or a comma separated list.
  • bb-varx-cut - Minimum X variance value for the BB cluster to be recommended instead of BBvarianceX
  • bb-vary-cut - Minimum Y variance value for the BB cluster to be recommended instead of BBvarianceY
  • ab-varx-cut - Minimum X variance value for the AB cluster to be recommended instead of ABvarianceX
  • ab-vary-cut - Minimum Y variance value for the AB cluster to be recommended instead of ABvarianceY
  • aa-varx-cut - Minimum X variance value for the AA cluster to be recommended instead of AAvarianceX
  • aa-vary-cut - Minimum Y variance value for the AA cluster to be recommended instead of AAvarianceY
  • bb-varx-z-cut - Minimum z-score X variance value for the BB cluster to be recommended instead of BBvarianceX
  • bb-vary-z-cut - Minimum z-score Y variance value for the BB cluster to be recommended instead of BBvarianceY
  • ab-varx-z-cut - Minimum z-score X variance value for the AB cluster to be recommended instead of ABvarianceX
  • ab-vary-z-cut - Minimum z-score Y variance value for the AB cluster to be recommended instead of ABvarianceY
  • aa-varx-z-cut - Minimum z-score X variance value for the AA cluster to be recommended instead of AAvarianceX
  • aa-vary-z-cut - Minimum z-score Y variance value for the AA cluster to be recommended instead of AAvarianceY
  • bb-varx-z-cut-enabled - Flag indicating if the Z thresholds should be on the BB X variances used instead of the variance thresholds with the supplemental workflow
  • bb-vary-z-cut-enabled - Flag indicating if the Z thresholds should be on the BB Y variances used instead of the variance thresholds with the supplemental workflow
  • ab-varx-z-cut-enabled - Flag indicating if the Z thresholds should be on the AB X variances used instead of the variance thresholds with the supplemental workflow
  • ab-vary-z-cut-enabled - Flag indicating if the Z thresholds should be on the AB Y variances used instead of the variance thresholds with the supplemental workflow
  • aa-varx-z-cut-enabled - Flag indicating if the Z thresholds should be on the AA X variances used instead of the variance thresholds with the supplemental workflow
  • aa-vary-z-cut-enabled - Flag indicating if the Z thresholds should be on the AA Y variances used instead of the variance thresholds with the supplemental workflow

If the SSPs file is turned on, ps-classification performs the supplemental workflow and the output the SSPs metrics along with the supplemental metrics:

  • use-ssp - Flag indicating if the SSP workflow should be run

If the biallelic metrics file contains metrics for special SNPs, the following classifiation thresholds are used with non-PAR X, Y, and MT SNPs:

  • hom-ro-hap-1-MTChr-cutoff - Minimum HomRO value for MT probesets with 1 haploid cluster
  • hom-ro-hap-2-MTChr-cutoff - Minimum HomRO value for MT probesets with 2 haploid clusters
  • homfld-YChr-cut - Minimum HomFLD_hap value for a Y probeset to be PHR instead of Other (male samples only)
  • min-YChr-samples-cut - Minimum number of male and female samples needed to classify Y probesets using the Euclidean distances thresholds
  • homfld-WChr-cut - Minimum HomFLD_hap value for a W probeset to be PHR instead of Other (female samples only)
  • min-WChr-samples-cut - Minimum number of male and female samples needed to classify W probesets using the Euclidean distances thresholds
  • hom-ro-hap-1-XChr-cutoff - Minimum HomRO value for non-PAR X probesets with 1 haploid cluster (male samples only)
  • hom-ro-hap-2-XChr-cutoff - Minimum HomRO value for non-PAR X probesets with 2 haploid clusters (male samples only)
  • homfld-XChr-cut - Minimum HomFLD_hap value for a non-PAR X probeset to be PHR instead of Other (male samples only)
  • fld-XChr-cut - Minimum FLD value for non-PAR X probesets (female samples only)
  • het-so-XChr-cutoff - Minimum HetSO value for non-PAR X probesets (female samples only)
  • aaf-XChr-cut - Maximum difference in A-allele frequency between female and male samples for non-PAR X probesets
  • hom-ro-hap-1-ZChr-cutoff - Minimum HomRO value for Z probesets with 1 haploid cluster (female samples only)
  • hom-ro-hap-2-ZChr-cutoff - Minimum HomRO value for Z probesets with 2 haploid clusters (female samples only)
  • homfld-ZChr-cut - Minimum HomFLD_hap value for a Z probeset to be PHR instead of Other (female samples only)
  • fld-ZChr-cut - Minimum FLD value for Z probesets (male samples only)
  • het-so-ZChr-cutoff - Minimum HetSO value for Z probesets (male samples only)
  • aaf-ZChr-cut - Maximum difference in A-allele frequency between female and male samples for Z probesets

If a multiallelic metrics file is provided, the following classification thresholds are used:

  • hom-mma-cutoff - Minimum Hom_MMA value needed to be recommended
  • fld-ma-cutoff - Minimum FLD_MA value needed to be recommended
  • fld-ma-2-cutoff - Minimum FLD_MA for probesets with 2 alleles
  • min-fld-ma-cutoff - Minimum FLD_MA value needed to be recommended
  • min-fld-ma-2-cutoff - Minimum FLD_MA for probesets with 2 alleles
  • het-so-ma-2-cutoff - Minimum HetSO_MA value needed for probesets wtih 2 alleles to be recommended
  • hom-ro-ma-cutoff - Minimum HomRO_MA value needed to be recommmended
  • hom-ro-ma-1-cutoff - Minimum HomRO_MA for probesets with 1 allele
  • hom-ro-ma-2-cutoff - Minimum HomRO_MA for probesets with 2 alleles
  • best-cr-cutoff - Minimum CR value needed to be recommended
  • genotype-p-value-cutoff - Minimum p-value for a probeset to be recommended instead of UnexpectedGenotypeFrequency

If a ps2snp-file is provided, then the best performance probeset will be selected to represent SNPs with multiple probesets. There are seven output files, one for each category, containing lists of probeset IDs belonging to each category. Each file contains a list of probeset IDs belonging to the category. The performance output file summarizes the categorization results across all SNPs.

If a psct-file is provided, then probeset-specific classification thresholds can be set.
