Introduction

apt-genotype-axiom is a program for making genotyping calls on the Affymetrix Axiom™ family of arrays. apt-genotype-axiom uses the BRLMM-P algorithm for genotype calling. BRLMM-P is a model-based approach which performs 1-dimensional clustering by fitting a Gaussian mixture model. Users wishing to use other legacy genotyping algorithms are recommended to use the apt-probeset-genotype application.

Quick Start

The command below runs the Axiom™ GT1 algorithm on Axiom™ arrays.

Note: The commands listed below are for standard biallelic analysis, without CNAG. For additional information on rare het adjustment, see FAQs below.

apt-genotype-axiom \
  --analysis-files-path /library/file/path \
  --arg-file Axiom_PMRA_96orMore_Step2.r3.apt-genotype-axiom.AxiomGT1.apt2.xml \
  --do-rare-het-adjustment true \
  --out-dir out \
  --cel-files cel_file_list.txt \
  --log-file ./apt-genotype-axiom.log

Alternatively, parameters can be set using command line parameters directly:

apt-genotype-axiom \
  --analysis-files-path /library/file/path \
  --analysis-name AxiomGT1 \
  --chip-type Axiom_PMRA.r3 \
  --do-rare-het-adjustment true \
  --artifact-reduction-clip 0.4 \
  --artifact-reduction-close 2 \
  --artifact-reduction-open 2 \
  --artifact-reduction-fringe 4 \
  --artifact-reduction-cc 2 \
  --sketch-target-scale-value 1000 \
  --sketch-size 50000 \
  --genotyping-node:brlmmp-CM 1 \
  --genotyping-node:brlmmp-bins 100 \
  --genotyping-node:brlmmp-mix 1 \
  --genotyping-node:brlmmp-bic 2 \
  --genotyping-node:brlmmp-lambda 1.0 \
  --genotyping-node:brlmmp-HARD 3 \
  --genotyping-node:brlmmp-SB 0.75 \
  --genotyping-node:brlmmp-transform MVA \
  --genotyping-node:brlmmp-copyqc 0.00000 \
  --genotyping-node:brlmmp-wobble 0.05 \
  --genotyping-node:brlmmp-MS 0.15 \
  --genotyping-node:brlmmp-copytype -1 \
  --genotyping-node:brlmmp-clustertype 2 \
  --genotyping-node:brlmmp-ocean 0.00001 \
  --genotyping-node:brlmmp-CSepPen 0.1 \
  --genotyping-node:brlmmp-CSepThr 4 \
  --genotyping-node:snp-priors-input-file Axiom_PMRA.r3.generic_prior.txt \
  --probeset-ids Axiom_PMRA.r3.step2.ps \
  --cdf-file Axiom_PMRA.r3.cdf \
  --special-snps Axiom_PMRA.r3.specialSNPs \
  --x-probes-file Axiom_PMRA.r3.chrXprobes \
  --y-probes-file Axiom_PMRA.r3.chrYprobes \
  --sketch-target-input-file Axiom_PMRA.r3.AxiomGT1.sketch \
  --igender-female-threshold 1.05 \
  --igender-male-threshold 1.1 \
  --allele-summaries true \
  --genotyping-node:snp-posteriors-output true \
  --out-dir out \
  --cel-files cel_file_list.txt \
  --log-file ./apt-genotype-axiom.log

Frequently Asked Questions

Question: Is there something I can do to improve the accuracy of rare heterozygous calls?

Answer: If you are genotyping a human array with batches of at least 90 samples, you may want to set the parameter --do-rare-het-adjustment TRUE. When this parameter is TRUE, then heterozygous calls for some probesets where the count of initial heterozygous calls is between 1 and --genotyping-node:max-rare-het-count will have additional tests applied. If the heterozygous call does not pass the additional tests, then it will be changed to NoCall.

Question: What are the additional output files if do-rare-het-adjustment is turned on?

Answer: AxiomGT1.rare_het.report.txt, which contains all (probeset, cel file) pairs in the batch for which the probeset was identified as having rare hets. It includes whether the call was changed to NoCall. The AxiomGT1.rare_het_maj_hom.report.txt contains homozygous cluster statistics for all probesets identified as having rare heterozygous calls.
