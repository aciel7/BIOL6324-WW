Introduction

ps-call-adjust` rewrites any genotype call to "No Call" where the call's confidence score fails a user-specified threshold, and outputs a calls file with the adjusted calls. To adjust only a given set of probesets, use a probeset list file with probeset_id as the first line. Quick Start

This command runs `ps-call-adjust` on the calls and confidences files in directory `./input` with the confidence threshold value of 0.05 and generates output files in the directory `./output`: ps-call-adjust --confidence-file ./input/AxiomGT1.confidences.txt --call-file ./input/AxiomGT1.calls.txt --confidence-threshold 0.05 --output-dir ./output

Details

`ps-call-adjust` output file is a modified version of the input calls file. All calls with matching confidences greater than the threshold will be rewritten to NoCall. The following parameters are required to run `ps-call-adjust`: -__calls-file__ - The file output from Axiom genotyping which contains the genotype calls (usually named "AxiomGT1.calls.txt"). -__confidences-file__ - The file output from Axiom genotyping which contains the genotype confidence scores (usually named "AxiomGT1.confidences.txt"). -__confidence-threshold__ - The value to compare the confidences against to decide if a call should be changed to "No Call". -__nocall-code__ - The numeric value to use for the new "No Call" genotype calls: default=-1.
