Introduction

`ps-extract` extracts data from calls, confidences, summary, references, posteriors (biallelic and multiallelic), and priors (biallelic and multiallelic) files for a supplied set of probesets and/or samples. Every input file has a matching output file. More details describing the use of `ps-extract` can be found in the

Quick Start

This command runs `ps-extract` on the calls and confidences files in directory `./input` and generate output files in the directory `./output`: ps-extract --confidence-file ./input/AxiomGT1.confidences.txt --call-file ./input/AxiomGT1.calls.txt --pid-file ./input/probesets.ps --output-dir ./output

Details

`ps-extract` output files are a modified, smaller version of the input files. The following parameters are required to run `ps-extract`:
  • --call-file-- The file output from Axiom genotyping which contains the genotype calls (usually named "AxiomGT1.calls.txt").
  • --confidence-file-- The file output from Axiom genotyping which contains the genotype confidence scores (usually named "AxiomGT1.confidences.txt").
  • --summary-file-- The file output from Axiom genotyping which contains intensity summaries for SNPs (usually named "AxiomGT1.summary.txt").
  • --posterior-file-- The file output from Axiom genotyping which contains posterior summaries of clusters for SNPs (usually named "AxiomGT1.snp-posteriors.txt").
  • --multi-posterior-file-- The file output from Axiom genotyping which contains multiallelic posterior summaries of clusters for SNPs (usually named "AxiomGT1.snp-posteriors.multi.txt").
  • --priors-file-- The file which contains prior summaries of clusters for SNPs (usually named "AxiomGT1.priors.txt").
  • --multi-priors-file-- The file which contains multiallelic prior summaries of clusters for SNPs (usually named "AxiomGT1.priors.multi.txt").
  • --references-file-- The file which contains the reference genotype calls (usually named "AxiomGT1.references.txt").
  • --output-dir-- Output directory for results files: defaults to current directory.
  • --pid-file-- The list of probesets that `ps-extract` should run on.
  • --sample-list-file-- The list of samples that `ps-extract` should run on.
